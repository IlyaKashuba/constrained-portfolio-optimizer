# Решатель задачи Марковица с реалистичными ограничениями
> Позволяет ввести данные по активам и получить структуру оптимального портфеля в смысле баланса риск-доходность

[![C#](https://img.shields.io/badge/C%23-10+-239120?style=flat&logo=csharp)](https://docs.microsoft.com/ru-ru/dotnet/csharp/)
[![.NET](https://img.shields.io/badge/.NET-6+-512BD4?style=flat&logo=dotnet)](https://dotnet.microsoft.com/)

 
## Задача
Реализовать алгоритм оптимизации портфеля ценных бумаг по модели Марковица с добавлением практических ограничений:
- размер позиций дискретный (k ценных бумаг i-го вида)
- стоимость портфеля не превышает размер капитала `B`
- размер коротких позиций не превышает `S*B`
- учитывается комиссия за покупку/продажу, комиссия = `Fee * |X - X_0|`, где `X` - искомый оптимальный портфель, `X_0` - портфель до ребалансировки, `Fee` - комиссия
- ценные бумаги заимствуются в шорт по ставке `r`
- для оптимизации портфеля используется функция полезности `U(X) = return(X) - risk(X)`

## Алгоритм решения
Задача решается генетическим алгоритмом, где
- каждая особь = состав портфеля
- функция приспособленности = функция полезности минус штрафы за нарушение ограничений

## Как использовать
Исходные данные для задачи вносятся в файл *data.json*:
- `N` и `G`- размер популяции и количество итераций для генетического алгоритма
- `n` - количество рассматриваемых активов
- `Prices` - массив цен на активы
- `Returns` - доходности активов
- `Covariances` - ковариации доходностей ценных бумаг
- `B` - размер капитала
- `Fee` - размер комиссии за покупку и продажу
- `OldPortfolio` - структура портфеля до ребалансировки
